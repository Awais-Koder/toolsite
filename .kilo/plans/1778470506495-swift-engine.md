# Professional UI/UX Redesign — Plan

## Current State Assessment

The site is a Laravel + TailwindCSS v4 + Alpine.js tool site (Time & Date Tools) with:
- `resources/views/layouts/app.blade.php` — master layout (header, main, footer)
- `resources/views/home.blade.php` — landing page with hero, how it works, features, FAQ, tools grid
- `resources/views/age-calculator.blade.php` — primary tool page (column layout, results, tabs, life story)
- `resources/views/pages/` — static pages: contact, about, privacy, terms, sitemap
- `resources/css/app.css` — Tailwind v4 theme tokens (colors, spacing, typography)
- `resources/js/app.js` — empty (only `//`)

**Why it feels basic/generic:**
- Lack of scroll/reveal animations (everything appears static)
- Very small border radii and flat shadows (not modern/professional)
- No interactivity polish on hover states beyond simple color/scale changes
- Mobile hamburger button is non-functional
- No entrance animations or staggered reveals
- Color palette is mostly monochrome (black + blue), lacks warmth/depth
- Typography hierarchy could be more expressive
- Blank `app.js` — no custom JS interactivity
- Static pages (contact, about) are extremely minimal with no visual interest

---

## Design Direction

- **Aesthetic:** Clean SaaS tool aesthetic — crisp whites, deep navy accents, generous whitespace, soft ambient gradients, glass-morphism touches.
- **Motion:** Smooth, subtle, purposeful. Every section should fade/slide into view on scroll. Cards should have elegant lift + glow on hover. Numbers should animate/c count up when they enter viewport.
- **Responsiveness:** Fully responsive with a polished mobile drawer nav.
- **Performance:** Use only CSS transitions + lightweight Intersection Observer (no heavy animation libraries).

## Implementation Plan

### Step 1: Design System Enhancement (`resources/css/app.css`)

Update the Tailwind `@theme` block with professional-grade tokens:

- **Border Radius:** Increase to modern values:
  - `--radius-default: 0.5rem`
  - `--radius-lg: 0.75rem`
  - `--radius-xl: 1rem`
  - `--radius-2xl: 1.5rem`
  - `--radius-full: 9999px`
- **Shadows:** Add a tiered shadow system (subtle → pronounced):
  - `@utility shadow-xs` — `0 1px 2px 0 rgb(0 0 0 / 0.03)`
  - `@utility shadow-sm` — `0 1px 3px 0 rgb(0 0 0 / 0.06), 0 1px 2px -1px rgb(0 0 0 / 0.04)`
  - `@utility shadow-md` — `0 4px 6px -1px rgb(0 0 0 / 0.06), 0 2px 4px -2px rgb(0 0 0 / 0.04)`
  - `@utility shadow-lg` — `0 10px 15px -3px rgb(0 0 0 / 0.07), 0 4px 6px -4px rgb(0 0 0 / 0.04)`
  - `@utility shadow-xl` — `0 20px 25px -5px rgb(0 0 0 / 0.08), 0 8px 10px -6px rgb(0 0 0 / 0.04)`
  - `@utility shadow-glow` — `0 0 40px -10px rgb(0 81 213 / 0.25)` (blue brand glow)
  - `@utility shadow-card` — `0 4px 20px -4px rgb(0 0 0 / 0.07)`
- **Animations / Transitions:**
  - Add custom easing curves: `--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1)`, `--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1)`
  - Add animation utilities for entrance animations:
    - `@utility animate-fade-up` — `opacity: 0; transform: translateY(24px);` (used as base, paired with JS toggled `.is-visible`)
    - `@utility animate-fade-in` — `opacity: 0;`
    - `@utility animate-scale-in` — `opacity: 0; transform: scale(0.96);`
    - Add transition utilities for `transition-all duration-500 ease-out-expo` etc.
- **Colors:** Keep existing since it's M3-like, but add a warm accent for featured badges/badges to break monochrome.
  - Add `--color-accent: #0f172a` (deep slate/navy) and `--color-accent-light: #e0e7ff`
  - Add `--color-success: #059669`
- **Background gradients:** Add utility `.bg-gradient-radial` for ambient blobs.

### Step 2: Global JavaScript Interactivity (`resources/js/app.js`)

Implement a lightweight animation engine:

1. **Scroll Reveal System:**
   - Use `IntersectionObserver` targeting elements with `[data-reveal]`.
   - Support data attributes: `data-reveal="fade-up"`, `data-reveal="fade-in"`, `data-reveal="scale-in"`, `data-reveal-delay="100"` (ms), `data-reveal-duration="600"`.
   - Add `.revealed` class when `threshold: 0.1` is crossed. CSS handles the actual transition via `transition` + final state classes.
   - Respect `prefers-reduced-motion`.

2. **Header Scroll Behavior:**
   - Track `window.scrollY`. When > 10px, add `is-scrolled` class to `<header>` that increases backdrop blur intensity, reduces border opacity, and potentially adds a subtle `shadow-sm`.

3. **Mobile Drawer Navigation:**
   - Alpine.js logic in `app.blade.php` will handle the drawer. Add a helper in `app.js` to lock/unlock body scroll when drawer opens (prevents background scroll).

4. **Number Counter Animation:**
   - On reveal of `[data-count-up]` elements, animate their `innerText` from 0 to target number over ~1500ms using `requestAnimationFrame` with `easeOutQuart`. Supports suffix/prefix via `data-prefix` and `data-suffix`.

5. **Ambient Background Gradients (optional subtle movement):**
   - Use CSS `@keyframes` for slow ambient movement of decorative gradient blobs (no JS needed).

### Step 3: Master Layout Redesign (`resources/views/layouts/app.blade.php`)

- **`<head>`:** Add a `<style>` block for base reveal styles (must be above Vite bundle to avoid FOUC), plus global CSS for smooth scrolling.
- **Header:**
  - Convert to glass-morphic style: `bg-white/80 backdrop-blur-xl` with a subtle bottom border `border-white/20`.
  - Add horizontal scroll-driven box-shadow transition when `is-scrolled` is active.
  - Logo: Add a subtle hover `scale(1.02)` and transition.
  - Mobile hamburger: Wire up with Alpine.js `x-data="{ open: false }"` and a full-screen or slide-over drawer.
    - Drawer should have staggered link reveal animations (each link fades in with increasing delay).
  - Nav links: Add an animated underline on hover (a pseudo-element that scales from 0 to 1 on X-axis with `transform-origin: left`). Active state keeps a persistent underline with a slight vertical offset (bottom border replacement).
- **Main:**
  - Increase `py-section-gap` to feel more spacious.
  - Add a subtle ambient background: absolute positioned radial gradient blobs behind main content (very low opacity, pointer-events-none, animated via CSS keyframes).
- **Footer:**
  - Redesign into a multi-column professional footer instead of the current single row.
  - Columns: Brand/Tagline, Tools (links), Company (About, Contact, Privacy, Terms), Social (placeholder icons).
  - Add a subtle top border with a gradient accent line.
  - Add a back-to-top button (appears after scroll) with smooth scroll behavior.
- **Back-to-top button:** Fixed bottom-right, appears after 400px scroll, glass-morphic circle.
